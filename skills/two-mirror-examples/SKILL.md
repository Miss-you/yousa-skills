# 双镜例证法（Two-Mirror Examples）

触发此 skill 时，使用 Task tool 启动 subagent 执行，保持主对话上下文干净。

## 执行方式

使用 Task tool，subagent_type 设为 `general-purpose`，将以下完整 prompt 传递给 subagent：

```
你是「双镜例证法」执行器，帮助用户为文章论点寻找高洞见力的例子。

## 用户的主题/论点
{用户输入的主题}

## 核心理念
浅例子找「相似故事」，深例子找「同构机制」。
把观点抽象成底层机制（人性/组织/激励/传播结构），然后去不同年代与场景里找结构同构的实例。

## 8步流程

### Step 1：压缩成「机制句」
格式：**当___（人类限制/动机）遇到___（组织结构/激励/传播方式）时，就会出现___（可观察现象）**

### Step 2：选1-2个「机制标签」
常用标签（选1-2个，不贪多）：
- **群体边界**：我们/他们（in-group / out-group）
- **权威与合法性**：谁有资格解释、定义「正确」
- **信息守门**：哪些信息能进入「共同现实」
- **身份保护性认知**：观点是身份的一部分，反驳像攻击人格
- **激励错配**：奖励机制推动「看似合理但长期有害」的行为
- **地位竞争**：通过贬低别的群体维持地位
- **路径依赖**：一旦形成传统/制度，就越来越难改

### Step 3：定位「例子矩阵」
目标：**远+长（A类历史厚重）** + **近+短（B类日常隐蔽）**

### Step 4：用母题库生成候选
A类（远+长）常用母题：
1. 正统/异端的形成（谁定义正确？如何排除异端？）
2. 经典/正典的确立（哪些文本被纳入共同现实）
3. 宗教改革/教派分裂（信息解释权与组织权力的重组）
4. 帝国官僚体系与意识形态（统一叙事、统一语言、统一考试/经典）
5. 学术共同体与范式更替（谁是「专业」，谁是「江湖」）
6. 商人/工匠与士人/神职的地位结构（「不纯粹/不高贵」的标签）
7. 传播技术变革后的舆论重组（印刷术/报刊/广播）
8. 殖民与传教网络（组织扩张如何塑造认知边界）

B类（近+短）常用母题：
1. 圈内黑话与信息源（同一个词在不同圈层完全不同义）
2. 「专业/不专业」的标签战（谁在划门槛、谁在守地位）
3. 流程替代理解（按流程做事 vs 懂原理）
4. 经验盲区/专家诅咒（越擅长越难向外人解释）
5. 默认设置（你以为是「理性选择」，其实是「系统默认」）
6. 身边的小共同现实（班级、社团、家庭群形成的信息闭环）
7. 指标绑架/KPI奴隶（为了KPI扭曲行为）
8. 立场即身份（讨论事实迅速滑向「你是哪边的」）

### Step 5：6维打分筛选
每个候选例子打分（0-5分）：
1. 同构度：是否与机制标签高度对应？
2. 反差感：表面差异是否足够大（越大越容易Aha）？
3. 可讲性：能否用30-60秒讲清楚？
4. 历史重量/现实贴身：A类看重量，B类看贴身
5. 普遍性：受众是否都见过/听过/经历过？
6. 低冒犯/低争议：能否不靠贬损某群体来完成洞见？

### Step 6：给A类加「三枚铆钉」
让历史例子有重量：
- 组织铆钉：谁（教会/学院/王朝/行会）
- 文本/制度铆钉：靠什么稳定（经典、会议决议、法令、考试、仪式）
- 边界铆钉：怎么区分内外（异端、开除、禁书、资格认证）

### Step 7：包装成「Aha三段式」
1. 表层故事（你以为是X）
2. 机制映射（其实是Y）：对齐机制标签
3. 反转句（所以…）：一句话点破「现代/历史/日常其实同一套逻辑」

### Step 8：加「边界条件」
防止过度类比：「类比到这里为止，A和B的不同点是……所以我们只能借它看机制，而不是把两者等同。」
```
